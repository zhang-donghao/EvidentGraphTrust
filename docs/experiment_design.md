# 实验与消融设计

本文档详细描述 Evident Graph Trust (EGT) 在物联网与车联网信任建模中的实验方案、对比基线、消融设置与评价指标，旨在满足经典学术论文对实验章节的撰写规范。

## 1. 数据集准备

### 1.1 VeReMi (Vehicular Reference Misbehavior Dataset)
- **任务目标**：识别车辆广播消息是否存在伪装、位置欺骗等恶意行为。
- **原始数据**：SUMO/Veins 仿真生成的 V2X 消息日志，包含车辆 ID、时间戳、位置、速度、方向、攻击标签。
- **构图策略**：
  1. 以车辆为节点；
  2. 若两个车辆在同一时间窗口内距离小于阈值、或存在 V2X 通信记录，则建立无向边；
  3. 边特征可选：相对速度、历史信任度差异、RSSI 强度（若可用）。
- **特征工程**：
  - 数值特征：位置差分、速度、加速度、航向角等；
  - 类别特征：攻击类型（训练时不可见，测试用于评估子类表现）；
  - 历史特征：基于滑动窗口统计的信誉得分、历史误差率。
- **数据划分**：按时间划分 60%/20%/20%（Train/Val/Test），保持攻击类型分布一致。

### 1.2 TON_IoT (Telemetry and Network data)
- **任务目标**：在多域 IoT 系统中检测被攻陷的设备节点。
- **原始数据**：来自 CCTV、环境传感器、智能网关等的遥测记录与网络流量，附带正常/攻击标签。
- **构图策略**：
  1. 将设备/服务实例建模为节点；
  2. 根据网络连接、协议交互或共同子网建立边；
  3. 边权可基于通信频次或延迟。
- **特征工程**：
  - 数值特征：传感器读数统计量、流量特征（包数、字节数等）；
  - 类别特征：协议类型、设备类别；
  - 历史行为：移动平均误差、告警次数。
- **数据划分**：按设备划分，确保测试集中包含未见过的设备以评估泛化能力。

## 2. 模型设置

### 2.1 我们的方法：EGT
- 主干：两层图卷积 (GCN/GAT) + 证据投影层；
- 证据输出：`e = softplus(W h + b)`，Dirichlet 参数 `alpha = e + 1`；
- 损失函数：
  - 数据项：采用 Dempster-Shafer evidential cross-entropy；
  - 正则项：KL(`Dir(alpha)` || `Dir(1)`) 约束先验；
  - 不确定性约束：根据置信度对错误预测施加惩罚；
- 先验嵌入：利用车辆信誉/设备类型生成先验证据向量。

### 2.2 基线模型
- **GCN/GAT/GraphSAGE**：保持相同层数与隐藏维度，输出概率分布；
- **MLP**：两层全连接网络，仅使用节点特征；
- **XGBoost**：基于特征的梯度提升树，使用 5 折交叉验证调参；
- **Evidential MLP**：无图结构的 evidential 版本，用于对比结构贡献。

所有模型使用相同的输入特征与标准化策略。GNN 模型采用 PyTorch Geometric 实现，传统模型使用 scikit-learn / XGBoost。

## 3. 消融实验

| 消融项 | 修改 | 目的 |
| ------ | ---- | ---- |
| 无证据正则 | 移除 KL 与退化惩罚，仅保留交叉熵 | 观察模型校准与不确定性变化 |
| 无先验嵌入 | 先验证据固定为均匀分布 | 验证领域知识贡献 |
| 无边结构 | 移除图边，退化为 Evidential MLP | 评估图结构重要性 |
| 单层图卷积 | 将两层 GNN 减为一层 | 检查网络深度对性能影响 |
| 温度缩放 | 在推理阶段加入/移除温度缩放 | 分析后处理对校准的作用 |

## 4. 评价指标与统计检验

- **准确率/召回率/F1**：衡量分类性能，宏平均处理类不平衡。
- **ROC-AUC & PR-AUC**：关注不同阈值下的区分能力，尤其是 PR-AUC 适用于不平衡数据。
- **ECE & MCE**：评估概率校准；配合可靠性图展示。
- **Brier Score**：整体概率误差。
- **Negative Log-Likelihood**：证据模型的对数似然表现。
- **攻击检测特定指标**：False Alarm Rate, Miss Detection Rate。
- **置信区间**：基于 5 次独立运行或 5 折划分，计算 95% 置信区间；如需检验显著性，使用成对 t 检验或 Wilcoxon 符号秩检验。

## 5. 训练配置

- 优化器：AdamW，学习率 1e-3（根据验证集调整）。
- 批量策略：采用全图训练或按子图采样（GraphSAGE）。
- Dropout：0.3；权重衰减：1e-4。
- 训练轮次：最大 200 epoch，验证指标无提升 20 epoch 后早停。
- 随机种子：设定 5 个固定种子（42, 2023, 4096, 777, 1024）。
- 硬件：单卡 RTX 3090（或类似），列出显存与训练时间。

## 6. 实验记录规范

- 使用 Weights & Biases 或 TensorBoard 记录损失、指标与校准曲线。
- 每次运行保存配置、随机种子、模型权重、评估结果 JSON。
- 代码中提供 `scripts/run_experiments.sh` 自动化执行主实验与消融。
- 在论文附录中提供表格列出超参数、运行时间、模型大小。

## 7. 论文撰写要点

- **数据集**：描述来源、预处理、划分比例、统计特征。
- **实验设置**：明确硬件、软件版本、超参数、重复次数。
- **主结果**：以表格展示各模型指标，突出 EGT 表现及置信区间。
- **消融分析**：单独表格/图表展示各消融项对性能与校准影响。
- **不确定性可视化**：可靠性图、置信度直方图、案例分析。
- **讨论**：结合场景解释 EGT 如何提升信任建模的可靠性，指出局限与未来工作。



## 8. 实验自动化脚本

使用 ``scripts/run_experiments.py`` 可以批量运行第 2 节列出的模型与第 3 节的消融组合：
```bash
python scripts/run_experiments.py \ 
  --dataset-root data \ 
  --dataset-name toni_iot \ 
  --epochs 100
```
脚本会在 ``runs/<dataset>/`` 下为每次实验创建文件夹，并生成：
- ``history.json``：记录训练/验证损失曲线，便于复现论文中的收敛图；
- ``test_metrics.json``：包含 Accuracy、F1、ROC-AUC、ECE、Brier 等指标；
- ``reliability.png``：可靠性图，可直接用于论文可视化；
- ``summary.json``：整合所有实验与随机种子的结果，方便在 notebook/Excel 中做对比统计。
如需单独运行某个模型，可调用 ``src/train.py`` 并使用 ``--model``/``--disable-evidence-regularizer`` 等参数覆盖默认设置。
